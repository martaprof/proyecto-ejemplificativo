# Evaluación

Esta carpeta recoge los instrumentos de evaluación: rúbricas, listas de cotejo, criterios, evidencias de aprendizaje y propuestas de calificación.

## Enfoque de evaluación

La evaluación se plantea como un proceso continuo y formativo. Debe valorar tanto el proceso seguido como el producto final obtenido.

Se evaluarán:

- investigación y planificación;
- cálculos y diseño de circuitos;
- simulaciones en Tinkercad;
- programación con Arduino;
- montaje y pruebas;
- documentación técnica;
- trabajo cooperativo;
- presentación final;
- reflexión y mejora.

## Instrumentos

- Rúbrica de proyecto.
- Lista de cotejo de seguridad y montaje.
- Revisión de ejercicios individuales.
- Observación sistemática en el aula.
- Revisión de simulaciones.
- Revisión del código.
- Memoria técnica.
- Exposición oral.
- Autoevaluación.
- Coevaluación.

## Relación orientativa con competencias específicas

| Competencia | Evidencias principales |
| --- | --- |
| CE1. Proyectos tecnológicos | Planificación, diseño, prototipado, memoria y exposición. |
| CE2. Materiales y sostenibilidad | Selección de componentes, seguridad y reflexión sobre uso de recursos. |
| CE3. Herramientas digitales | Simulaciones en Tinkercad, código Arduino y documentación digital. |
| CE4. Saberes científicos | Cálculos eléctricos y análisis de circuitos. |
| CE5. Programación y automatización | Lectura de sensores, control de actuadores y comportamiento del sistema. |
| CE6. Energía y sostenibilidad | Análisis del consumo, eficiencia y aplicación tecnológica al invernadero. |

## Momentos de evaluación

| Momento | Qué se evalúa |
| --- | --- |
| Inicio | Comprensión del reto y planificación inicial. |
| Desarrollo | Ejercicios, simulaciones, código, colaboración y revisión. |
| Integración | Montaje, funcionamiento y resolución de problemas. |
| Cierre | Producto final, memoria, exposición y reflexión. |
