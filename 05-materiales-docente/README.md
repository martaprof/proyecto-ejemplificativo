# Materiales para el docente

Esta carpeta incluye orientaciones internas para el profesorado: notas de implementación, posibles soluciones, consejos de dinamización, gestión del aula y alternativas.

## Orientaciones generales

El proyecto debe plantearse como un reto abierto, pero con andamiaje suficiente para que el alumnado de 1.º de Bachillerato pueda avanzar. La clave es evitar que el alumnado se limite a copiar un montaje, haciendo que cada bloque técnico responda a una necesidad del sistema del invernadero.

## Decisiones didácticas importantes

- Definir desde el inicio qué variables se van a medir: temperatura, luminosidad y humedad simulada.
- Presentar el invernadero como contexto realista, no como decorado del proyecto.
- Mantener visible la pregunta guía durante todo el proceso.
- Entregar la rúbrica antes de iniciar el desarrollo técnico.
- Alternar explicaciones breves con tareas de aplicación directa.
- Usar Tinkercad como espacio de prueba antes del montaje físico.
- Reservar tiempo para depuración y mejora, no solo para construir.

## Posibles niveles de guía

| Nivel | Descripción |
| --- | --- |
| Guiado | El docente proporciona esquemas base y código parcialmente iniciado. |
| Intermedio | El alumnado completa cálculos, ajusta umbrales y modifica el código. |
| Abierto | Cada equipo decide umbrales, indicadores, estructura del sistema y mejoras. |

## Dificultades previsibles

- Confusión entre magnitudes analógicas y digitales.
- Errores de conexión en protoboard.
- Interpretación incorrecta de sensores.
- Dificultades para relacionar código y circuito.
- Falta de reparto equilibrado de tareas en los equipos.

## Estrategias de apoyo

- Usar simulaciones parciales antes de integrar el sistema completo.
- Proporcionar ejemplos mínimos de lectura de sensores.
- Revisar primero el esquema antes de permitir el montaje físico.
- Pedir que cada equipo documente errores y soluciones.
- Introducir roles: coordinación, montaje, programación, documentación y pruebas.
