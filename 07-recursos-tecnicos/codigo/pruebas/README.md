# Códigos mínimos de prueba

Esta carpeta recoge programas sencillos para comprobar cada componente por separado antes de integrarlo en el prototipo final.

| Archivo | Componente o función |
| --- | --- |
| [`01-led-parpadeo.ino`](01-led-parpadeo.ino) | Prueba de un LED con resistencia. |
| [`02-ldr-monitor-serie.ino`](02-ldr-monitor-serie.ino) | Lectura de LDR por monitor serie. |
| [`03-tmp36-monitor-serie.ino`](03-tmp36-monitor-serie.ino) | Lectura del TMP36 y conversión a grados Celsius. |
| [`04-potenciometro-humedad.ino`](04-potenciometro-humedad.ino) | Lectura de potenciómetro como humedad simulada. |
| [`05-zumbador.ino`](05-zumbador.ino) | Prueba básica de zumbador. |
| [`06-servo-basico.ino`](06-servo-basico.ino) | Movimiento básico de servomotor. |
| [`07-servo-dos-ldr.ino`](07-servo-dos-ldr.ino) | Control del servomotor con dos LDR. |

## Uso recomendado

Antes del montaje final, cada equipo debería ejecutar las pruebas mínimas de los componentes que vaya a utilizar. Esto permite separar errores de conexión, errores de código y fallos de integración.
