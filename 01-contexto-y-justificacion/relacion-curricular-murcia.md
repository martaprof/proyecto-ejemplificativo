# Relación curricular

## Referencia normativa

Decreto n.º 251/2022, de 22 de diciembre, por el que se establece la ordenación y el currículo de Bachillerato en la Comunidad Autónoma de la Región de Murcia. (2022). Boletín Oficial de la Región de Murcia, (296), pp. 6755-6778. [https://www.borm.es/eli/es-mc/d/2022/12/22/251/dof/spa/html](https://www.borm.es/eli/es-mc/d/2022/12/22/251/dof/spa/html)

## Materia y etapa

- **Etapa:** Bachillerato.
- **Curso:** 1.º de Bachillerato.
- **Materia:** Tecnología e Ingeniería I.
- **Comunidad autónoma:** Región de Murcia.

## Competencias específicas relacionadas

| Competencia específica | Relación con el proyecto |
| --- | --- |
| CE1. Coordinación y desarrollo de proyectos tecnológicos | El alumnado diseña, planifica, simula, documenta y presenta un sistema electrónico aplicado a un invernadero. |
| CE2. Selección de materiales y sostenibilidad | Se seleccionan componentes electrónicos y se reflexiona sobre alimentación, eficiencia, uso de recursos y aplicación en un contexto sostenible. |
| CE3. Uso de herramientas digitales | Se emplean Tinkercad, Arduino IDE o el entorno de programación de Tinkercad para simular, programar y documentar el sistema. |
| CE4. Aplicación de saberes científicos y técnicos | Se aplican cálculos de resistencias, divisores de tensión, regulación, lectura de sensores y análisis de circuitos. |
| CE5. Programación y automatización | Se programan lecturas analógicas, umbrales de decisión, avisos y control de servomotor. |
| CE6. Uso responsable y eficiente de la energía | Se analiza la alimentación del sistema, la posible integración de energía solar y la relación entre tecnología y sostenibilidad. |

## Criterios de evaluación vinculados

| Criterio | Aplicación en el proyecto |
| --- | --- |
| 1.1 | Investigación y diseño de una solución tecnológica para monitorizar condiciones de un invernadero. |
| 1.2 | Mejora iterativa del prototipo a partir de simulaciones, pruebas y correcciones. |
| 1.3 | Trabajo cooperativo mediante roles, planificación y reparto de tareas. |
| 1.4 | Elaboración de esquemas, memoria técnica, tablas de prueba y documentación del código. |
| 1.5 | Presentación oral del producto final y explicación técnica de la solución. |
| 2.1 | Análisis del funcionamiento del sistema, calidad del prototipo y propuestas de mejora. |
| 2.2 | Selección razonada de sensores, actuadores, resistencias, reguladores y elementos de conexión. |
| 2.3 | Simulación y montaje de prototipos respetando normas básicas de seguridad. |
| 3.1 | Uso de Tinkercad y Arduino para diseñar, programar, simular y validar el sistema. |
| 3.2 | Elaboración de presentaciones y documentación digital del proyecto. |
| 4.2 | Resolución de problemas de corriente continua, resistencias, divisores, regulación y lectura analógica. |
| 5.1 | Programación básica de un sistema electrónico con sensores y salidas. |
| 5.2 | Automatización de respuestas mediante avisos y, opcionalmente, servomotor. |
| 5.3 | Explicación secuencial del funcionamiento del programa. |
| 6.1 | Análisis del consumo, alimentación y eficiencia del sistema. |
| 6.2 | Reflexión sobre sostenibilidad, uso responsable de la tecnología y aplicación a un invernadero. |

## Saberes básicos relacionados

| Bloque | Relación con el proyecto |
| --- | --- |
| Proyectos de investigación y desarrollo | Planteamiento del reto, planificación, prototipado, documentación y presentación. |
| Materiales y fabricación | Selección de componentes, interpretación de hojas de características y montaje. |
| Sistemas eléctricos y electrónicos | Resistencias, divisores de tensión, Zener, transistores, comparadores, lógica digital y sensores. |
| Sistemas informáticos y programación | Programación con Arduino, lectura de entradas, salidas digitales y monitor serie. |
| Sistemas automáticos | Relación sensor-controlador-actuador y control del servomotor. |
| Tecnología sostenible | Uso eficiente de recursos, alimentación de baja tensión y posible integración de energía solar. |

## Evidencias de evaluación

- Simulaciones en Tinkercad.
- Código Arduino.
- Ejercicios de cálculo.
- Esquemas eléctricos.
- Registro de errores y pruebas.
- Prototipo físico o simulado.
- Memoria técnica.
- Presentación final.
- Autoevaluación y coevaluación.
